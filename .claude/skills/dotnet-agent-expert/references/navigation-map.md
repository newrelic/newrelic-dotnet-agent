# Navigation Map

A growing set of **example** question families, each showing where a topic lives
in the live source and — crucially — the **non-obvious trap** it hides. These are
illustrative, **not** the boundary of answerable questions: most questions will have
no entry here, and that is expected. When no entry matches, answer anyway using the
**completeness sweep in `SKILL.md` step 4**, which generalizes the trap-shapes these
entries encode (second/parallel path, snapshot-vs-incremental, scope/destination
filter, per-property precedence, one-input-silences-another, non-default destination,
exclusive/asymmetric bound). If you uncover a trap worth pre-recording, surface it to the **user** as a
one-line maintainer note at the end of your answer; only add an entry here if
the user asks (the skill is read-only at answer time). Entries are never a fence
on what may be asked.

Anchors are **coarse on purpose** — directory + class + grep string, never line
numbers — so the *anchors* resist staleness. But the **trace-note facts and any
"Verified live" notes are point-in-time** and must be re-confirmed against the
live tree before you repeat them. An entry's **Answers:** list bounds THAT
entry's scope, not the topic's — when a question reaches past the listed
Answers, the entry is exhausted; run the completeness sweep against the
question's real scope. Always grep the search terms against the
current tree and read the code before answering; generate citations live. Lean on
the repo's `CLAUDE.md` sub-docs as the first-stop orientation layer
(`src/claude-source.md`, `tests/claude-tests.md`, `docs/config-development.md`) and
add only the question-family anchors they don't already cover.

## Collector / region selection
**Answers:** How does the agent decide whether to connect to the US vs EU (or other region) collector?

## Log levels
**Answers:** What logging levels does the agent support, and where are they defined/parsed?
**Start in repo docs:** `src/claude-source.md` §"Debugging signals" (mentions `NEWRELIC_LOG_LEVEL=debug`); root `CLAUDE.md` (debug logs to `<home>/logs/`)

**Search terms:** `MapToSerilogLogLevel`, `IsLogLevelDeprecated`, `AuditLevel`, `TranslateLogLevel`
**Trace notes:** The documented set is on the `log/@level` attribute in `Configuration.xsd` (default `info`): off, error, warn, info, debug, finest, all. `LogLevelExtensions.MapToSerilogLogLevel` parses the string case-insensitively onto Serilog levels and accepts many deprecated aliases (verbose/fine/finer/trace/notice/alert/critical/emergency/fatal/severe) that still work but warn. `OFF` maps above Fatal (disables logging); `AUDIT` is not a level (treated as INFO with a warning — audit logging is enabled via the `auditLog` attribute). `TranslateLogLevel` converts back to the FINEST/DEBUG/INFO/WARN/ERROR log-file labels. **Key class(es):** `ErrorService`/`IErrorService`, `ErrorData`, `ExceptionFormatter`, `DefaultConfiguration.ParseExpectedErrorConfigurations`/`ParseIgnoreErrorConfigurations`/`ParseExpectedStatusCodesArray`, `MatchRule` + `StatusCodeExactMatchRule` + `StatusCodeInRangeMatchRule`, `ServerConfiguration.RpmConfig`
**Search terms:** `FromExceptionInternal`, `GetBaseException`, `IsExceptionSpecified`, `IsErrorFromExceptionSpecified`, `ContainsSubstring`, `GetFriendlyExceptionTypeName`, `ExpectedErrorsConfiguration`, `IgnoreErrorsConfiguration`, `ExpectedStatusCodes`, `ShouldIgnoreHttpStatusCode`, `FromErrorHttpStatusCode`, `ParseExpectedStatusCodesArray`, `StatusCodeInRangeMatchRule`, `NEW_RELIC_ERROR_COLLECTOR_EXPECTED_ERROR_CODES`, `NEW_RELIC_ERROR_COLLECTOR_IGNORE_ERROR_CODES`, `GetEnvironmentVariableFromList`, `Enumerable.Empty`

All matching lives in `ErrorService`; rule sets are built once in the `DefaultConfiguration.Parse*` methods and exposed as `ExpectedErrorsConfiguration` / `IgnoreErrorsConfiguration` (both `IDictionary<string, IEnumerable<string>>` keyed by friendly type/class name or status-code string) and `ExpectedStatusCodes` (`IEnumerable<MatchRule>`). **GOTCHA — split field provenance:** in `FromExceptionInternal` the error TYPE name and MESSAGE come from `exception.GetBaseException()` (innermost real cause) while the STACK TRACE is `ExceptionFormatter.FormatStackTrace` over the OUTERMOST exception (and recurses the whole `InnerException` chain) — deliberate per a code comment, so type/message and stack trace can describe different frames. **GOTCHA — message match is substring, same for both:** `ContainsSubstring` is `String.Contains`; configured messages are substrings, not exact and not regex, and expected + ignored both funnel through `IsExceptionSpecified`, so "expected uses regex / ignored uses exact" is wrong for type/message. **GOTCHA — the real asymmetry is STATUS CODES:** expected codes parse into `MatchRule` objects (`StatusCodeExactMatchRule`/`StatusCodeInRangeMatchRule`, integer compare) so expected supports RANGES like `400-404`; ignored codes have NO range support — folded as keys into `IgnoreErrorsConfiguration` and matched by `ContainsKey(formattedCode)` with a major-code fallback (split on the period). **GOTCHA — empty message list = match the class regardless of message:** `IsExceptionSpecified` does `if (messages != Enumerable.Empty<string>())` then substring-matches, ELSE returns true — REFERENCE equality against the cached `Enumerable.Empty<string>()` singleton, and the parser deliberately assigns exactly that singleton for class entries and ignore-status-codes, so only those take the match-all branch. Friendly type name (`GetFriendlyExceptionTypeName`) strips the generic-arity backtick suffix, so config keys must use the non-generic `FullName`. ## High Security Mode (HSM) vs LASP security policies
**Answers:** "Why aren't my custom attributes / custom events / custom parameters captured?" "Why is `record_sql` forced to obfuscated even though I set raw?" "Why does the agent refuse to connect / shut down with a security-policies error?" "What's the difference between high security mode and security policies (LASP), and which one is overriding my config?"
**Start in repo docs:** root `CLAUDE.md` §Configuration (precedence) points to `Core/Configuration` and `Core/Config`; HSM/LASP specifics are not called out (none directly).
**Directory:** `src/Agent/NewRelic/Agent/Core/Configuration/` and `src/Agent/NewRelic/Agent/Core/DataTransport/`
**Key class(es):** `DefaultConfiguration`, `SecurityPoliciesConfiguration`, `SecurityPolicy`, `BoolConfigurationItem`/`RecordSqlConfigurationItem`, `SecurityPoliciesValidationException`, `SecurityPolicyState`, `PreconnectResult`, `SecuritySettingsModel`/`SecurityPoliciesSettingsModel` (in `ConnectModel.cs`), `ConnectionHandler`/`ConnectionManager`, `ConfigurationService`
**Search terms:** `HighSecurityModeEnabled`, `HighSecurityModeOverrides`, `HighSecurityConfigSource`, `SecurityPolicyConfigSource`, `KnownPolicies`, `SecurityPoliciesTokenExists`, `security_policies_token`, `ValidateNotBothHsmAndSecurityPolicies`, `GetMissingExpectedSeverPolicyNames`, `GetMissingRequiredPolicies`, `SecurityPoliciesValidationException`, `ApplyIfMoreRestrictive`, `CustomEventsPolicyName`
**Trace notes:** Two independent mechanisms converge in `DefaultConfiguration`. **(1) HSM is purely LOCAL:** `HighSecurityModeEnabled` reads `highSecurity.enabled` (env override `NEW_RELIC_HIGH_SECURITY`) and hard-overrides in-process — bool getters short-circuit `if (HighSecurityModeEnabled) return new BoolConfigurationItem(false, HighSecurityConfigSource)` BEFORE consulting policy/local/server values (custom attributes, custom params, custom events, custom-instrumentation-editor), which is why those silently vanish under HSM regardless of `newrelic.config`. `record_sql` is forced to `obfuscated` (raw never allowed) by seeding a `RecordSqlConfigurationItem(ObfuscatedStringValue, HighSecurityConfigSource)` and ratcheting via `ApplyIfMoreRestrictive` (most-restrictive wins: off > obfuscated > raw — it only ratchets DOWN). **(2) LASP is SERVER-NEGOTIATED:** config key `securityPoliciesToken` (env `NEW_RELIC_SECURITY_POLICIES_TOKEN`); `ConnectionHandler.SendPreconnectRequest` sends it as `security_policies_token` and the server returns `security_policies` (`PreconnectResult.SecurityPolicies` → `Dictionary<string, SecurityPolicyState{enabled, required}>`). **GOTCHA — provenance, not bare bools:** HSM/LASP results are `BoolConfigurationItem`/`RecordSqlConfigurationItem` objects carrying a Source string (`"High Security Mode"`/`"Security Policy"`/`"Local Configuration"`/`"Server Configuration"`) exposed via paired `...Source` properties, so support logs can show WHO disabled a setting (`LogSecurityPolicySettingsOnceAllSettingsResolved`). **GOTCHA — fixed `KnownPolicies` / refuse-to-connect:** exactly six policy names (`record_sql`, `attributes_include`, `allow_raw_exception_messages`, `custom_events`, `custom_parameters`, `custom_instrumentation_editor`); `ConnectionHandler` runs four validations during Connect — `ValidateNotBothHsmAndSecurityPolicies` (HSM + token together throws), `ValidateAgentTokenSettingToPoliciesReceived` (token-without-policies or vice versa throws), `ValidateAgentExpectedSecurityPoliciesExist` (`GetMissingExpectedSeverPolicyNames` — note the source's "Sever" typo), and `ValidateAllRequiredPoliciesFromServerExist` (`GetMissingRequiredPolicies` — server sent `required=true` for a policy this agent version doesn't know → throw); all throw `SecurityPoliciesValidationException`. **GOTCHA — that exception is FATAL, not retryable:** it isn't a network exception, so `ConnectionManager.HandleConnectionException` hits the `default` arm, sets `shouldRestart=false`, and calls `ImmediateShutdown()` (`KillAgentEvent`) — the agent stops rather than backing off. On success, policies flow via `SecurityPoliciesConfigurationUpdatedEvent` → `ConfigurationService` rebuild. The connect payload echoes resolved settings two ways: `SecuritySettingsModel` (always present) carries the HSM/general `record_sql` as a bare string via `TransactionTraceSettingsModel` and does NOT validate it; `SecurityPoliciesSettingsModel` (only added when `SecurityPoliciesTokenExists`) serializes each policy as a `{enabled:bool}` dict and its CONSTRUCTOR throws `ArgumentException` if `record_sql` resolved to raw (`ConnectModel.cs`, `SecurityPoliciesSettingsModel` ctor). So the raw→throw guard is on the LASP policies model, not the HSM settings model. **Search terms:** `TraceparentPresent`, `ApplyRemoteParentSampledBehavior`, `AcceptDistributedTraceHeaders`, `NumberOfFieldsV0`, `NumberOfFieldsInSupportedVersion`, `SamplerLevel`, `isRemoteParentSampled`, `GetSamplerType`, `NEW_RELIC_DISTRIBUTED_TRACING_SAMPLER_REMOTE_PARENT_SAMPLED`
**Trace notes:** Origin: `Transaction.AcceptDistributedTraceHeaders` → `DistributedTracePayloadHandler.AcceptDistributedTraceHeaders` → static `TracingState.AcceptDistributedTraceHeaders<T>(carrier, getter, transportType, trustedAccountKey, startTime, ISamplerService)`. **GOTCHA — `traceparent` silences the `newrelic` header (even when invalid):** the `newrelic`-header probe/decode block is wrapped in `if (!TraceparentPresent)` with the source comment "if traceparent was present (regardless if valid), ignore newrelic header." `TraceparentPresent` is set true on PRESENCE before parsing (in `W3CTraceContext.TryGetTraceParentHeaderFromHeaders`), so a malformed traceparent (which records `TraceParentParseException`) still suppresses the legacy header. **Nuance:** "present" means EXACTLY ONE `traceparent` value — `result.Count() != 1` (zero or multiple) returns null/absent, and THEN the `newrelic` header IS read (probing three case variants `newrelic`/`NEWRELIC`/`Newrelic`). **W3C validity (`W3CTraceparent.GetW3CTraceParentFromHeader`):** rejects length < 55 or < 4 dash-fields; version is 2 lowercase-hex chars (`ff` invalid); a v0 header requires EXACTLY 4 fields; `TraceId` 32-hex non-zero, `ParentId` 16-hex non-zero, `TraceFlags` 2-hex; `Sampled` = bit 0. Tracestate NR entry (`{trustedAccountKey}@nr`) needs exactly 9 fields at v0 (> 9 for future versions), a `ParentType` in App..Mobile, non-empty account/app ids, parseable timestamp; `Sampled` only if it parses to 0/1, `Priority` ≤ 6 decimal places. **Sampling decision + remote-parent override (`ApplyRemoteParentSampledBehavior`):** `isRemoteParentSampled` comes from `Traceparent.Sampled` (when present) else the NR payload's `Sampled`; if both null it returns (no remote parent). Otherwise it picks `SamplerLevel.RemoteParentSampled` or `RemoteParentNotSampled`, calls `ShouldSample(SamplingParameters(...))`, and assigns `_priority`/`_sampled` — overriding the getters' fallback to tracestate/payload values. **GOTCHA — dead null-guard:** `ApplyRemoteParentSampledBehavior` has an `if (sampler == null) return;` and `SamplerService.GetSampler`'s XML-doc claims it returns null for "Default" behavior, but `InitializeSamplers` always populates all three levels (Default → `Adaptive`), `SamplerFactory.GetSampler` never returns null, and `GetSampler` otherwise THROWS `ArgumentOutOfRangeException` — so that guard is effectively dead and "default" remote-parent behavior runs through the `AdaptiveSampler`. **`AdaptiveSampler.ShouldSample` honors upstream:** it early-returns the inbound tracestate `Sampled`/`Priority` (when NR trace context accepted & `Tracestate.Sampled.HasValue`), then the inbound NR payload `Sampled`, and only otherwise makes its own adaptive decision (`TracePriorityManager.Adjust`, `PriorityBoost=1.0`). `AlwaysOn` → `(true, 2.0)` and `AlwaysOff` → `(false, 0.0)` unconditionally (so e.g. `remote_parent_sampled=alwaysOff` overrides an upstream `sampled=true`); `TraceIdRatioBased` decides from the FIRST 16 hex chars of `TraceId`. **GOTCHA — Root sampler skipped on override:** `Transaction.Sampled` returns `TracingState.Sampled` when `HasValue` (which itself falls back to tracestate/payload `Sampled`), and `Transaction.SetSampled` runs the Root sampler ONLY inside `if (!Sampled.HasValue)` — so any inbound/overridden sampled value makes `Sampled.HasValue` true and the Root branch is skipped, leaving the upstream/remote-parent decision standing. **Config:** `DefaultConfiguration.GetSamplerType` resolves each level — env (`NEW_RELIC_DISTRIBUTED_TRACING_SAMPLER_ROOT`/`_REMOTE_PARENT_SAMPLED`/`_REMOTE_PARENT_NOT_SAMPLED`) wins, else `newrelic.config` `sampler.root`/`remoteParentSampled`/`remoteParentNotSampled`, else legacy; null maps to `Adaptive`.
