# Plan: Add ability to disable ConfigurationManager API support (NR-323920)

## Context
The agent reads several configuration values (AppName, Labels, LicenseKey, AgentEnabled, config file path) via `System.Configuration.ConfigurationManager` on .NET Framework and via `Microsoft.Extensions.Configuration` bridging on .NET Core. When the application being monitored has hooked into ConfigurationManager (e.g., custom config providers), this can cause deadlocks during agent initialization. The fix adds a flag that, when set, causes the agent to skip all ConfigurationManager reads and rely solely on newrelic.config and environment variables.

## Naming conventions
- **XML attribute**: `disableConfigurationManagerSupport` on `<service>` element (default: `false`)
- **Environment variable**: `NEW_RELIC_DISABLE_CONFIGURATION_MANAGER_SUPPORT`
- **IConfiguration property**: `bool DisableConfigurationManagerSupport { get; }`

Add attribute to `configurationService` complex type, after `sendDataOnExit`:
```xml
<xs:attribute name="disableConfigurationManagerSupport" type="xs:boolean" default="false">
    <xs:annotation>
        <xs:documentation>
            If true, the agent will skip all ConfigurationManager API reads. Use this to
            prevent deadlocks when the monitored app has configured ConfigurationManager to
            integrate with APIs the agent also instruments. Any settings previously read
            from web.config/app.config/appsettings.json must be migrated to newrelic.config
```

### 5. Add env var check to ConfigurationManagerStatic constructors
**`src/Agent/NewRelic/Agent/Core/Configuration/IConfigurationManagerStatic.cs`**

Both the NETFRAMEWORK and the else branch (NETSTANDARD2_0) `ConfigurationManagerStatic` classes get a constructor that pre-disables via env var:
```csharp
public ConfigurationManagerStatic()
{
    if (IsDisabledViaEnvironmentVariable())
        localConfigChecksDisabled = true;
}

private static bool IsDisabledViaEnvironmentVariable()
{
    var val = System.Environment.GetEnvironmentVariable("NEW_RELIC_DISABLE_CONFIGURATION_MANAGER_SUPPORT");
    return bool.TryParse(val, out var b) ? b : val == "1";
}
```
This handles bootstrap-time reads (before `DefaultConfiguration` exists) for both platforms.

### 7. Guard BootstrapConfiguration AppSettings read
**`src/Agent/NewRelic/Agent/Core/Config/BootstrapConfiguration.cs`**

In `SetAgentEnabledValues()`, skip `TryGetAgentEnabledFromAppSettings()` when disabled:
```csharp
private void SetAgentEnabledValues()
{
    _agentEnabledWithProvenance = TryGetAgentEnabledFromWebConfig();
    if (_agentEnabledWithProvenance != null) return;

    if (!ConfigLoaderHelpers.IsConfigurationManagerSupportDisabled())
    {
        _agentEnabledWithProvenance = TryGetAgentEnabledFromAppSettings();
        if (_agentEnabledWithProvenance != null) return;
    }

    _agentEnabledWithProvenance = new ValueWithProvenance<bool>(_agentEnabledValueFromLocalConfig, ConfigurationFileName);
}
```
